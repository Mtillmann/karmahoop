karmahoop

How does it work?

The five-character-code is permutated to each of the 32 possibly combinations of upper- and lowercase chars. Then the corresponding image's thumbnail is loaded from imgur and it is checked whether or not the thumbnail is 160pixels in height. A lower height of 81pixel indicates that the request was 404 because thats the height of imgur's error picture.

r1, r2, f, g+?

Those links point to reddit submission links. 1 is the raw image url, 2 is the embedded view on imgur.com (no "i" subdomain and no file extension). If the image has been submitted to reddit already you could find the related discussion. Since imgur.com does not care about the original extension (I only use .jpg) it is possible that the raw url link may be wrong because the image was uploaded and submitted with another file extension (ie .gif, .png). If this is the case you are in luck and will be able to repost the image for great justice!

The f and the g+ link let you post the image to facebook or google+.

The first link goes to the image's embedded imgur.com view where you can download the image in full resolution or request deletion of the image.

What kind of Server is needed?

None! You can download the whole thing from github, unpack and run it from your disk. Or host a copy it on your own server.

Images and JSONP?

I tried to access the images' response codes through jsonp. It works but there is no error handling for 404 and requests that are valid throw an error as they contain binary image data but not valid javascript/json. It would have been nice also to be able to use the HEAD method for more compact requests and responses but that also does not work as expected...

Where does the content come from?

All images are loaded from imgur.com. Imgur is image sharing site that is heavily used by the community of reddit.com. All images have been submitted by users of imgur.com and are not preselected in any way. If you find offensive material please report it to imgur.com.

Is this a good idea?

Propably not. I hope imgur.com does not take damage from this brute force method of snooping. The actual hit ratio per code is between 0% and 20%. For every image you see there are 5-15 requests that fail.
